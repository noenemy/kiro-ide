# Lab 1. Kiro IDE로 이커머스 웹서비스 만들기
## 개요 
Kiro IDE의 채팅창에 자연어로 지시해서 쉽고 빠르게 서비스를 만들 수 있습니다.
이 랩에서는 온라인 쇼핑 데이터를 관리하는 웹서비스를 만들어 봅니다. 데이터베이스는 DynamoDB로, 웹서비스는 로컬에서 호스팅 합니다.

AWS에 대한 지식이 없더라도 자연어로 서비스를 구현 가능합니다.

**Let's go!**

---

## 사전 준비
- [Kiro IDE 설치](https://kiro.dev/docs/getting-started/installation/) 완료
- AWS CLI 설정 완료 (AWS 계정 연결)
- Kiro IDE에서 워크스페이스 폴더 열기

> [!WARNING]
> 로컬에 Kiro IDE 를 설치해서 실습할 경우 AWS CLI 에 구성되어 있는 연결계정으로 사용할 수 있습니다. 또한 로컬의 파일과 환경을 변경할 수 있습니다.

---

## Step 1. Kiro IDE 채팅창에서 자연어로 대화하기

Kiro IDE를 실행하고, 좌측 채팅 패널을 엽니다.

* 다음과 같이 입력해서 샘플 이커머스 고객 데이터를 만들어 봅니다.
```
10개의 샘플 이커머스 고객의 데이터를 만들어 주세요. 
```
출력이 어떤 모습인가요? 다른 동료와 출력 결과를 확인해 보세요.

* 이번에는 출력 결과를 명시적으로 간단하게 하기 위해 지시 내용에 다음과 같이 추가해 실행해 봅니다.
```
10개의 샘플 이커머스 고객의 데이터를 csv 형식으로 만들어 주세요. 
```
이전 출력과 어떻게 달라졌는지 확인해주세요. 여러분이 스스로 주문을 추가해보시기 바랍니다.

* 출력 형식을 한국어로 지정하기 위해 다음과 같이 채팅창에 입력해 봅니다.
```
모든 답변은 한국어로 작성해주시되, 기술 용어는 영어로 표기해주세요.
```

* Kiro IDE가 지시사항을 따르는지 다음과 같이 테스트 합니다.
```
이커머스 데이터를 관리하는 웹 애플리케이션을 만드는 가장 간단한 방법은 무엇인가요? 방법론만 설명해주세요
```
답변이 위의 지시사항 처럼 한국어로, 기술용어는 영어로 표기되었는지 확인합니다.

---

## Step 2. Steering으로 일관된 응답 받기

Kiro IDE에서는 **Steering** 파일을 통해 AI에게 일관된 지시사항을 전달할 수 있습니다.
Kiro CLI의 Knowledge 기능과 유사하지만, 프로젝트 내 `.kiro/steering/` 폴더에 마크다운 파일로 관리합니다.

* 채팅창에 다음과 같이 입력하여 Steering 파일을 생성합니다.
```
.kiro/steering/ 폴더에 ecommerce.md 라는 Steering 파일을 만들어주세요. 내용은 다음과 같습니다:

- 응답 시 지시사항을 명시적으로 참조해주세요.
- AWS를 능숙하게 다루는 이커머스 서비스 개발자의 관점에서 답변해주세요.
- 서비스 구축은 최대한 간단하고 신속하게 진행해주세요.
- 모든 답변은 한국어로 작성해주시되, 기술 용어는 영어로 표기해주세요.
- 추측이 필요한 경우 반드시 이를 명시해주세요.
- AWS 리소스 작업 시 us-east-1 리전을 사용해주세요.
- DynamoDB의 describe_table의 ItemCount는 신뢰할 수 없으므로 사용을 제한해주세요.
```

> [!TIP]
> Steering 파일은 `.kiro/steering/` 폴더에 위치하며, 기본적으로 모든 채팅에 자동 적용됩니다.
> 특정 파일 패턴에만 적용하려면 front-matter에 `inclusion: fileMatch`와 `fileMatchPattern`을 설정할 수 있습니다.
> 수동으로 적용하려면 `inclusion: manual`로 설정하고, 채팅에서 `#` 컨텍스트 키로 참조합니다.

* Steering 파일이 생성되었는지 확인합니다. 좌측 파일 탐색기에서 `.kiro/steering/ecommerce.md` 파일을 열어봅니다.

> [!NOTE]
> Kiro CLI에서는 `/knowledge add` 명령어로 지식을 추가했지만, Kiro IDE에서는 Steering 파일이 자동으로 인식됩니다.
> 별도의 활성화 명령 없이 `.kiro/steering/` 폴더에 파일을 두면 바로 적용됩니다.

## Step 3. DynamoDB 데이터베이스 만들기

이제 DynamoDB를 이용해서 데이터베이스를 만들겠습니다. Kiro IDE 채팅창에 다음과 같이 지시합니다.

```
DynamoDB를 이용해서 이커머스 데이터베이스를 만들어주세요. 
테이블은 총 4개입니다. 1) Categories 2) Products 3) Customers 4) Orders
제품에는 에어콘과 난방기를 포함해주세요.
각 테이블마다 샘플 데이타는 테이블당 50개 이하로 만들어주세요.
```

> [!NOTE]
> Kiro IDE는 Autopilot 모드에서 터미널 명령어를 자동으로 실행할 수 있습니다.
> Supervised 모드에서는 각 명령어 실행 전에 확인을 요청합니다.
> 좌측 하단에서 현재 모드를 확인하고, 필요시 전환할 수 있습니다.

이렇게 DynamoDB에 대한 지식이 없는 누구나 쉽게 DynamoDB를 이용하여 샘플 데이타를 만들 수 있습니다.
Steering 적용 전후, Kiro IDE의 응답에 변화가 있었나요? 어떤 것인가요?

* DynamoDB에 테이블 4개가 모두 생성되었는지, 각 테이블마다 샘플 데이타가 모두 지시대로 생성되었는지 확인해 봅니다. 혹시 누락된 것이 있다면 채팅창에서 직접 지시해서 보완합니다.
```
DynamoDB에 테이블 4개가 모두 생성되었는지, 각 테이블마다 샘플 데이타가 있는지 확인해주세요. 
```
Kiro IDE가 지시받은 대로 작업을 했는지 확인합니다.
* 4개의 테이블과 샘플 데이타가 모두 생성되었는지 확인합니다.
* 샘플 데이타가 모두 생성되었는지 확인합니다.
* AWS 콘솔에서 DynamoDB 콘솔로 이동하여 직접 확인할 수도 있습니다.

---

## Step 4. 이커머스 웹서비스 만들기

위에 생성한 데이터베이스에 입출력과 보기 기능을 제공하는 웹서비스를 만들어 보겠습니다. 빠르게 시스템을 구축하기 위해 다음과 같이 채팅창에 입력합니다.

```
지금 있는 DynamoDB를 이용해서 이커머스 데이터베이스를 관리할 수 있는 웹서비스를 로컬에 만들어 주세요.
다음 조건들을 준수해주시기 바랍니다:
1. Python 내장 HTTP 서버(http.server)만 사용해주세요
2. 모든 UI는 단일 HTML 파일로 구현해주세요
3. JavaScript와 CSS는 HTML 파일 내에 포함해주세요
4. 모든 DynamoDB 테이블에 대해 CRUD 기능을 모두 구현해주세요
5. 모든 DynamoDB 테이블에 대해 간단한 검색 기능을 포함해주세요
6. localhost로 접속 가능하도록 서버를 바인딩하세요
```

<BR>

>[!NOTE]
>웹서비스 구축 파일 생성에는 시간이 15분 이상 소요될 수 있습니다. 시간 절약을 위해 Kiro IDE가 이미 구현한 웹서비스 파일을 활용해보세요.
>
>* [ecommerce_server.py](ecommerce_server.py) 
>* [ecommerce.html](ecommerce.html) 
<BR>

Kiro IDE는 파일을 자동으로 생성하고 에디터에 표시합니다. 생성된 코드를 직접 확인할 수 있습니다.

* 완료가 되었습니까? 혹시 중간에 끊기거나 완료되지 않으면 `계속하세요` 라고 입력합니다.

* 완료가 되었다면 웹서비스의 포트를 확인합니다. (기본 포트는 8000)

---

## Step 5. 웹서비스 실행 및 확인

완료가 되었다면 Kiro IDE의 터미널에서 웹서비스를 실행합니다.

> [!TIP]
> Kiro IDE 터미널은 상단 메뉴 `Terminal > New Terminal` 또는 단축키 `` Ctrl+` ``(macOS: `` Cmd+` ``)로 열 수 있습니다.

```bash
python3 ecommerce_server.py
```

브라우저에서 `http://localhost:8000` 으로 접속합니다.

* 접속을 할 수 있습니까? 접속이 되지 않는다면 Kiro IDE 채팅창에 질의합니다.
```
당신이 위에서 만들어 준 웹서비스로 접속이 되지 않는 원인을 점검해 주세요. 
```
* 웹서비스로 접속을 할 수 있게 되면 페이지를 둘러봅니다. 함께 랩을 진행한 동료의 웹페이지와 차이점을 비교해 봅니다.
* 웹페이지의 기능이 요구사항 대로 동작하는지 검증해 보세요.

---

## Step 6. 요구사항 변경 대응하기

시스템 구축 단계에서 요구사항이 변경된다고 가정합니다. 우리 시스템에는 `리뷰`라는 항목이 추가되어야 합니다. 데이터베이스와 웹서버 모두 변경이 필요합니다. Kiro IDE 채팅창에 요구사항을 입력합니다.

```
리뷰라는 항목이 추가되어야 합니다. 웹서비스와 다이나모 디비 모두 리뷰라는 항목이 추가되도록 변경해 주세요.
```

> [!TIP]
> Kiro IDE는 기존 파일을 직접 수정합니다. 변경 사항은 에디터에서 diff로 확인할 수 있으며,
> Supervised 모드에서는 변경 적용 후 되돌릴 수 있습니다.

Kiro IDE가 데이터베이스와 웹서비스를 모두 변경하는지 검수합니다.

* 완료된 후에 웹사이트를 둘러보고 기능을 점검합니다.
* 변경하고 싶은 부분을 생각해 봅니다.
* 여러분 만의 요구사항을 자연어로 지시합니다.

## Kiro CLI vs Kiro IDE 비교

| 항목 | Kiro CLI | Kiro IDE |
|------|----------|----------|
| 실행 방식 | 터미널에서 `kiro-cli` 실행 | IDE 실행 후 채팅 패널 사용 |
| 지시사항 관리 | Knowledge (`/knowledge add`) | Steering (`.kiro/steering/*.md`) |
| 파일 생성/수정 | 터미널에서 파일 생성 | 에디터에서 직접 생성/수정 + diff 확인 |
| 코드 확인 | `cat`, `vi` 등 터미널 명령어 | 에디터에서 바로 확인 + 구문 강조 |
| 실행 모드 | 단일 모드 | Autopilot / Supervised 모드 선택 |
| MCP 설정 | `~/.kiro/settings/mcp.json` 직접 편집 | `.kiro/settings/mcp.json` 또는 UI에서 설정 |
| 컨텍스트 공유 | 파일 경로 지정 | `#File`, `#Folder`, 이미지/문서 드래그 |
