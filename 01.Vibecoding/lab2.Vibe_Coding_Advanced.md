# Lab 2. 바이브코딩으로 이커머스 서비스 고도화하기

## 개요
Lab 1에서 자연어 한 마디로 이커머스 웹서비스를 만들어 보았습니다.
이번 랩에서는 Kiro IDE의 **Steering**과 **Hooks** 기능으로 가드레일을 설정한 뒤, 바이브코딩으로 서비스를 대폭 고도화합니다.

이 랩을 통해 다음을 체험합니다:
- Steering으로 코딩 품질의 기준선을 잡는 방법
- Hooks로 자동 검증 워크플로우를 구축하는 방법
- Vibecoding으로 UI/UX를 극적으로 개선하는 방법
- 프롬프트를 점점 정교하게 다듬는 과정

## Step 1. Steering으로 코딩 가드레일 설정하기

Kiro의 **Steering**은 AI에게 프로젝트의 핵심 규칙이나 컨텍스트를 지속적으로 학습시키는 Markdown 기반의 설정 파일입니다. 이를 통해 매 대화마다 번거롭게 규칙을 설명할 필요 없이, AI가 우리 팀의 코딩 스타일이나 아키텍처 가이드를 항상 인지하도록 만들 수 있습니다.

Steering 문서의 주요 특징
- 저장 위치: 프로젝트 루트 디렉터리의 .kiro/steering/ 폴더 안에 .md 파일 형식으로 저장합니다.
- 작동 방식: 이 폴더 내의 Markdown 파일들은 AI와의 모든 상호작용 시 자동으로 컨텍스트에 포함되어 가이드라인 역할을 합니다.
- 사용 목적: 프로젝트의 아키텍처, 코딩 컨벤션, 자주 사용하는 라이브러리 사용법, 비즈니스 로직 정의 등을 기록하여 일관된 코드 생성을 유도합니다. 

Lab 1에서는 응답 언어와 역할에 대한 Steering만 설정했습니다.
이번에는 **코드 품질**에 대한 Steering을 추가하여, 바이브코딩으로 생성되는 코드의 수준을 높여보겠습니다.

Kiro IDE 채팅창에 다음과 같이 입력합니다.


.kiro/steering/ 폴더에 coding-standards.md 라는 Steering 파일을 만들어주세요. 내용은 다음과 같습니다:

```
## 코딩 표준

### Python
- 모든 함수에 docstring을 작성할 것
- 에러 핸들링은 try/except로 구체적인 예외를 잡을 것 (bare except 금지)
- 로깅은 print 대신 logging 모듈을 사용할 것
- 함수는 하나의 역할만 수행하도록 분리할 것

### HTML/CSS/JavaScript
- CSS는 BEM 네이밍 컨벤션을 따를 것
- JavaScript는 const/let만 사용하고 var는 금지
- 모든 버튼과 입력 필드에 접근성 속성(aria-label)을 포함할 것
- 색상은 CSS 변수로 관리하여 테마 변경이 용이하도록 할 것

### 공통
- 주석은 "왜(why)"를 설명하고, "무엇(what)"은 코드로 표현할 것
- 매직 넘버 금지, 상수로 정의할 것
```

생성된 파일을 열어서 내용을 확인합니다.

> [!TIP]
> Steering 파일은 여러 개를 만들 수 있습니다. 예를 들어 `ecommerce.md`는 도메인 지식, `coding-standards.md`는 코딩 규칙으로 분리하면 관리가 편합니다.

## Step 2. Hooks로 자동 검증 설정하기

 Kiro의 Hook은 IDE에서 특정 이벤트가 발생하면 AI 에이전트가 자동으로 작업을 수행하는 
  자동화 기능입니다.                                                                  
                                                                                      
  ### 주요 트리거                                                                     
                                                                                      
  | 트리거 | 발동 시점 | 활용 예시 |                                                  
  |--------|----------|----------|                                                    
  | File Save | 파일 저장 시 | 린팅, 테스트 자동 실행, 문서 업데이트 |                
  | File Create | 새 파일 생성 시 | 보일러플레이트 생성, 테스트 파일 자동 생성 |      
  | Agent Stop | 에이전트 응답 완료 시 | 생성된 코드 컴파일/포맷팅 |                  
  | Manual Trigger | 수동 실행 | 온디맨드 코드 리뷰, 보안 스캔 |                      
                                                                                      
  ### 생성 방법                                                                       
                                                                                      
  1. Kiro 패널에서 **Agent Hooks** → **+** 클릭                                       
  2. 자연어로 설명하면 Kiro가 Hook을 자동 생성                                        
                                                                                      
  예: *"React 컴포넌트 저장할 때마다 테스트 파일 자동 업데이트해줘"*                  
                                                                                      
  ### 특징                                                                            
                                                                                      
  - Git에 커밋하면 팀 전체에 동일한 품질 기준 적용                                    
  - 액션 타입: **Ask Kiro** (AI 프롬프트) 또는 **Run Command** (셸 명령어)            
  - 자세한 내용: [Kiro Hooks 문서](https://kiro.dev/docs/hooks/) 


바이브코딩으로 코드를 생성할 때마다 자동으로 품질을 검증하는 Hook을 만들어 보겠습니다.

### Hook 1: Python 파일 저장 시 구문 검사

채팅창에 다음과 같이 입력합니다.

```
Python 파일(.py)이 수정될 때마다 자동으로 python3 -m py_compile 명령어로 구문 검사를 실행하는 Hook을 만들어주세요.
```

> [!NOTE]
> Hook은 `.kiro/hooks/` 폴더에 JSON 파일로 저장됩니다.
> 좌측 탐색기의 **Agent Hooks** 섹션에서 현재 등록된 Hook을 확인할 수 있습니다.

### Hook 2: 코드 변경 시 리뷰 요청

채팅창에 다음과 같이 입력합니다.

```
파일 쓰기 작업이 완료된 후, 변경된 코드가 .kiro/steering/coding-standards.md에 정의된 코딩 표준을 준수하는지 자동으로 리뷰하는 Hook을 만들어주세요.
```

이제 바이브코딩으로 코드를 생성하면:
1. 파일이 저장될 때 자동으로 구문 검사가 실행되고
2. 코드 작성 후 자동으로 코딩 표준 준수 여부를 리뷰합니다

> [!TIP]
> Hook이 너무 자주 실행되어 불편하다면, 좌측 탐색기의 Agent Hooks 섹션에서 개별 Hook을 비활성화할 수 있습니다.

## Step 3. 관리자 대시보드 추가하기

이제 가드레일이 설정되었으니, 본격적으로 바이브코딩을 시작합니다.
Lab 1에서 만든 이커머스 서비스에 **관리자 대시보드**를 추가해 보겠습니다.

채팅창에 다음과 같이 입력합니다.

```
현재 이커머스 웹서비스에 관리자 대시보드 페이지를 추가해주세요.

대시보드에는 다음 내용을 포함해주세요:
1. 총 주문 수, 총 매출액, 총 고객 수, 총 상품 수를 카드 형태로 표시
2. 카테고리별 상품 수를 파이 차트로 표시
3. 최근 주문 목록 (최신 10건)
4. 인기 상품 Top 5

Chart.js CDN을 사용해서 차트를 그려주세요.
대시보드는 기존 웹서비스의 네비게이션에 "Dashboard" 메뉴로 추가해주세요.
```

Kiro IDE가 코드를 생성하는 과정을 관찰합니다:
* Steering에서 정의한 코딩 표준이 적용되고 있나요?
* Hook이 자동으로 실행되나요?
* 생성된 코드에 docstring이 포함되어 있나요?

완료되면 터미널에서 서버를 재시작하고 대시보드를 확인합니다.

```bash
python3 ecommerce_server.py
```

브라우저에서 `http://localhost:8000` 에 접속하여 대시보드 메뉴를 클릭합니다.

## Step 4. UI/UX 대폭 개선하기

Lab 1에서 만든 기본 UI를 모던한 디자인으로 완전히 탈바꿈시켜 보겠습니다.
바이브코딩의 진가는 "자연어 한 마디로 얼마나 극적인 변화를 만들 수 있는가"에 있습니다.

채팅창에 다음과 같이 입력합니다.

```
현재 이커머스 웹서비스의 UI를 전면 개편해주세요.

디자인 요구사항:
1. 모던하고 깔끔한 디자인 (Tailwind CSS CDN 사용)
2. 다크 모드 / 라이트 모드 전환 토글 버튼
3. 좌측 사이드바 네비게이션 (아이콘 포함)
4. 테이블은 줄무늬(striped) 스타일, 호버 시 하이라이트
5. 각 CRUD 작업에 성공/실패 토스트 알림
6. 상품 목록은 카드 그리드 뷰와 테이블 뷰를 전환할 수 있게
7. 모바일 반응형 지원

기존 기능(CRUD, 검색)은 모두 유지해주세요.
```

> [!NOTE]
> 이 프롬프트는 상당히 큰 변경을 요청합니다. Kiro IDE가 작업하는 동안 에디터에서 파일이 실시간으로 변경되는 것을 관찰해 보세요.
> Supervised 모드에서는 각 변경 사항을 확인하고 되돌릴 수 있습니다.

완료되면 서버를 재시작하고 결과를 확인합니다.

* 다크 모드 전환이 동작하나요?
* 카드 뷰 / 테이블 뷰 전환이 되나요?
* 모바일 크기로 브라우저를 줄여보세요. 반응형이 적용되었나요?
* 동료의 결과물과 비교해 보세요. 같은 프롬프트인데 결과가 다른가요?

## Step 5. 프롬프트 다듬기 — 이미지로 컨텍스트 전달하기

바이브코딩에서 원하는 결과를 얻으려면 프롬프트를 점점 구체적으로 다듬어야 합니다.
Kiro IDE는 **이미지를 채팅에 첨부**할 수 있어서, 원하는 디자인을 시각적으로 전달할 수 있습니다.

### 방법 1: 텍스트로 수정 요청

결과물에서 마음에 들지 않는 부분을 구체적으로 지적합니다.

```
대시보드의 카드 레이아웃을 수정해주세요.
- 카드 4개를 한 줄에 나란히 배치 (모바일에서는 2x2 그리드)
- 각 카드에 아이콘 추가 (주문: 📦, 매출: 💰, 고객: 👥, 상품: 🏷️)
- 카드에 전일 대비 증감률 표시 (임의 데이터로)
- 카드 호버 시 살짝 위로 올라오는 애니메이션
```

### 방법 2: 이미지로 디자인 전달

원하는 디자인의 스크린샷이나 목업 이미지가 있다면, 채팅창에 드래그하여 첨부합니다.

```
(이미지 첨부 후)
이 이미지와 비슷한 스타일로 대시보드를 수정해주세요.
```

> [!TIP]
> Kiro IDE 채팅창에 이미지를 전달하는 방법:
> - 채팅 입력창 좌측의 📎 아이콘 클릭
> - 이미지 파일을 채팅창으로 드래그 앤 드롭
> - 클립보드에서 붙여넣기 (스크린샷 후 Cmd+V)

## Step 6. 컨텍스트 활용하기 — #File, #Folder

Kiro IDE에서는 `#` 키를 사용하여 특정 파일이나 폴더를 컨텍스트로 전달할 수 있습니다.
이 기능을 활용하면 바이브코딩의 정확도를 높일 수 있습니다.

### 기존 코드를 참조한 수정 요청

채팅창에서 `#`을 입력하면 파일/폴더 목록이 나타납니다.

```
#ecommerce_server.py 이 파일의 에러 핸들링을 개선해주세요.
모든 API 엔드포인트에 try/except를 추가하고, 
에러 발생 시 적절한 HTTP 상태 코드와 JSON 에러 메시지를 반환하도록 해주세요.
```

### 문제 진단에 활용

```
#Problems 현재 파일에서 발생하는 문제들을 모두 수정해주세요.
```

### Git 변경사항 확인

```
#Git Diff 지금까지 변경된 내용을 요약해주세요.
```

> [!TIP]
> 사용 가능한 컨텍스트 키:
> - `#File` — 특정 파일을 컨텍스트로 전달
> - `#Folder` — 폴더 전체를 컨텍스트로 전달
> - `#Problems` — 현재 파일의 문제점
> - `#Terminal` — 터미널 출력 내용
> - `#Git Diff` — Git 변경사항

## Step 7. 자유롭게 Vibecoding 하기

지금까지 배운 기능들을 활용하여 자유롭게 이커머스 서비스를 개선해 보세요.

아래는 시도해 볼 만한 프롬프트 예시입니다:

**기능 추가**
```
장바구니 기능을 추가해주세요. 상품 목록에서 "장바구니 담기" 버튼을 누르면 
우측 상단에 장바구니 아이콘과 담긴 수량이 표시되고, 
장바구니 페이지에서 수량 변경과 삭제가 가능해야 합니다.
```

**데이터 시각화**
```
주문 데이터를 기반으로 월별 매출 추이를 라인 차트로 보여주는 페이지를 추가해주세요.
```

**검색 고도화**
```
현재 검색 기능을 개선해주세요. 
검색어 입력 시 실시간으로 결과가 필터링되고, 
검색 결과에서 검색어가 하이라이팅 되도록 해주세요.
```

* 동료와 서로 다른 기능을 추가해 보고 결과를 비교해 보세요.
* 같은 요구사항이라도 프롬프트를 어떻게 작성하느냐에 따라 결과가 달라지는지 확인해 보세요.
* Steering과 Hooks가 코드 품질에 어떤 영향을 미치는지 관찰해 보세요.

## 정리

이번 랩에서 체험한 Kiro IDE 기능들을 정리합니다.

| 기능 | 역할 | 실습에서 활용한 방법 |
|------|------|---------------------|
| **Steering** | AI의 행동 규칙 정의 | 코딩 표준을 정의하여 생성 코드 품질 향상 |
| **Hooks** | IDE 이벤트 기반 자동화 | 파일 저장 시 구문 검사, 코드 리뷰 자동 실행 |
| **이미지 첨부** | 시각적 컨텍스트 전달 | 원하는 디자인을 이미지로 전달 |
| **#컨텍스트** | 특정 파일/정보 참조 | #File, #Problems, #Git Diff 활용 |
| **Autopilot/Supervised** | 자율성 수준 조절 | 대규모 변경 시 Supervised로 안전하게 진행 |

### 핵심 교훈
- **바이브코딩은 프롬프트가 전부입니다.** 구체적이고 명확한 프롬프트가 좋은 결과를 만듭니다.
- **Steering은 가드레일입니다.** 코딩 표준을 미리 정의하면 바이브코딩의 품질이 올라갑니다.
- **Hooks는 안전망입니다.** 자동 검증으로 실수를 조기에 발견할 수 있습니다.
- **같은 프롬프트, 다른 결과.** AI 생성 코드는 매번 다를 수 있으므로, 검증하는 습관이 중요합니다.
