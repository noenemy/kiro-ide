### 시나리오 1: 개인 포트폴리오 웹사이트 만들기

**목표:** HTML/CSS/JS로 자기소개 포트폴리오 사이트를 만든다.

**실습 프롬프트 예시:**
> "개인 포트폴리오 웹사이트를 만들어줘. 상단에 네비게이션 바, Hero 섹션에 이름과 한 줄 소개, Skills 섹션, Projects 섹션, Contact 폼이 있어야 해. 반응형으로 만들어줘."

**기대 결과물:**
- `index.html`, `style.css`, `script.js`
- 반응형 레이아웃 (모바일/데스크톱)
- 스크롤 애니메이션 또는 부드러운 스크롤 네비게이션

**실습 포인트:**
- Kiro가 생성한 HTML 구조를 확인하고 섹션 순서 변경 요청해보기
- "다크 모드 토글 추가해줘" 같은 후속 프롬프트로 기능 확장
- CSS 색상이나 폰트를 자연어로 변경 요청해보기

**확장 아이디어:**
- GitHub Pages로 배포
- 프로젝트 섹션에 실제 GitHub 레포 링크 연결

---

### 시나리오 2: 할 일 관리 앱 (Todo App)

**목표:** CRUD 기능이 있는 Todo 앱을 localStorage로 구현한다.

**실습 프롬프트 예시:**
> "할 일 관리 앱을 만들어줘. 할 일 추가, 완료 체크, 삭제 기능이 있고, 새로고침해도 데이터가 유지되도록 localStorage를 사용해줘. 필터 기능(전체/완료/미완료)도 넣어줘."

**기대 결과물:**
- 할 일 추가/삭제/완료 토글
- localStorage 기반 데이터 영속성
- 필터링 UI

**실습 포인트:**
- "할 일에 우선순위(높음/중간/낮음) 추가해줘" 후속 요청
- "드래그 앤 드롭으로 순서 변경 가능하게 해줘" 난이도 올리기
- Kiro가 생성한 코드에서 버그를 일부러 만들고 수정 요청해보기

**확장 아이디어:**
- 카테고리별 분류
- 마감일 설정 + 알림

### 시나리오 3: 가위바위보 게임

**목표:** 컴퓨터와 대결하는 가위바위보 게임을 만든다.

**실습 프롬프트 예시:**
> "가위바위보 게임을 만들어줘. 사용자가 버튼으로 선택하면 컴퓨터가 랜덤으로 선택하고, 결과를 보여줘. 전적(승/패/무)도 표시해줘. 이모지로 가위✌️ 바위✊ 보🖐️ 표시해줘."

**기대 결과물:**
- 3개 버튼 (가위/바위/보)
- 컴퓨터 랜덤 선택 로직
- 승패 판정 + 전적 카운터

**실습 포인트:**
- "5판 3선승제로 바꿔줘" 규칙 변경 요청
- "애니메이션 효과 추가해줘" UI 개선
- 게임 로직 코드를 읽고 이해해보기

**확장 아이디어:**
- 묵찌빠 확장
- 온라인 대전 (WebSocket)

### 시나리오 4: 랜덤 명언 생성기

**목표:** 버튼을 누르면 랜덤 명언을 보여주는 앱을 만든다.

**실습 프롬프트 예시:**
> "랜덤 명언 생성기를 만들어줘. 한국어 명언 20개를 포함하고, 버튼을 누르면 랜덤으로 하나를 보여줘. 배경색도 명언마다 바뀌게 해줘. 트위터 공유 버튼도 넣어줘."

**기대 결과물:**
- 명언 데이터 배열
- 랜덤 선택 + 배경색 전환
- SNS 공유 기능

**실습 포인트:**
- "명언을 외부 API에서 가져오도록 바꿔줘" 로 API 연동 학습
- "페이드 인/아웃 애니메이션 추가해줘"
- 명언 데이터를 JSON 파일로 분리 요청

**확장 아이디어:**
- 카테고리별 명언 (동기부여, 사랑, 유머)
- 즐겨찾기 기능

### 시나리오 5: 단위 변환기

**목표:** 온도, 길이, 무게 등 다양한 단위를 변환하는 도구를 만든다.

**실습 프롬프트 예시:**
> "단위 변환기를 만들어줘. 탭으로 온도(섭씨/화씨/켈빈), 길이(cm/m/inch/feet), 무게(kg/lb/oz)를 전환할 수 있게 해줘. 입력하면 실시간으로 변환 결과가 나오게 해줘."

**기대 결과물:**
- 탭 기반 카테고리 전환
- 실시간 양방향 변환
- 깔끔한 입력/출력 UI

**실습 포인트:**
- "환율 변환 탭도 추가해줘" 확장 요청
- 변환 공식이 맞는지 직접 검증해보기
- "변환 히스토리를 아래에 표시해줘"

**확장 아이디어:**
- 데이터 용량 변환 (KB/MB/GB/TB)
- 시간대 변환

### 시나리오 6: 디지털 시계 + 스톱워치

**목표:** 현재 시간 표시와 스톱워치 기능을 가진 앱을 만든다.

**실습 프롬프트 예시:**
> "디지털 시계와 스톱워치를 만들어줘. 상단에 현재 시간(시:분:초)을 크게 표시하고, 아래에 스톱워치(시작/정지/리셋/랩 기록) 기능을 넣어줘. 다크 테마로 만들어줘."

**기대 결과물:**
- 실시간 시계 (setInterval)
- 스톱워치 (시작/정지/리셋/랩)
- 다크 테마 UI

**실습 포인트:**
- "카운트다운 타이머도 추가해줘" 기능 확장
- setInterval과 clearInterval 동작 원리 이해
- "랩 기록을 테이블로 보여줘"

**확장 아이디어:**
- 알람 기능
- 세계 시계 (여러 타임존)

### 시나리오 7: BMI 계산기

**목표:** 키와 몸무게를 입력하면 BMI를 계산하고 결과를 시각적으로 보여준다.

**실습 프롬프트 예시:**
> "BMI 계산기를 만들어줘. 키(cm)와 몸무게(kg)를 입력하면 BMI 수치와 판정(저체중/정상/과체중/비만)을 보여줘. 결과에 따라 색상이 다르게 표시되고, 게이지 바로 시각화해줘."

**기대 결과물:**
- 입력 폼 (키, 몸무게)
- BMI 계산 + 판정 로직
- 색상 코딩된 결과 + 게이지 바

**실습 포인트:**
- BMI 계산 공식 검증
- "BMI 히스토리를 차트로 보여줘" 확장
- 입력 유효성 검사 추가 요청

**확장 아이디어:**
- 체지방률 계산기
- 목표 체중까지 필요한 칼로리 계산

### 시나리오 8: 색상 팔레트 생성기

**목표:** 랜덤 색상 조합을 생성하고 복사할 수 있는 도구를 만든다.

**실습 프롬프트 예시:**
> "색상 팔레트 생성기를 만들어줘. 스페이스바를 누르면 5개의 조화로운 색상 조합이 생성되고, 각 색상을 클릭하면 HEX 코드가 클립보드에 복사되게 해줘. 색상 잠금 기능도 넣어줘."

**기대 결과물:**
- 5색 팔레트 생성
- 클릭 시 HEX 복사
- 색상 잠금/해제 토글
- 키보드 단축키 (스페이스바)

**실습 포인트:**
- 색상 조화 알고리즘 이해 (보색, 유사색 등)
- "CSS 변수로 내보내기 기능 추가해줘"
- "그라데이션 미리보기도 보여줘"

**확장 아이디어:**
- 이미지에서 색상 추출
- Tailwind/CSS 코드 자동 생성

### 시나리오 9: 마크다운 미리보기 에디터

**목표:** 왼쪽에 마크다운을 입력하면 오른쪽에 실시간 미리보기가 나오는 에디터를 만든다.

**실습 프롬프트 예시:**
> "마크다운 에디터를 만들어줘. 화면을 좌우로 나눠서 왼쪽에 마크다운 입력, 오른쪽에 실시간 HTML 미리보기를 보여줘. 코드 블록 구문 강조, 테이블 지원도 해줘. 다운로드 버튼으로 .md 파일 저장도 가능하게."

**기대 결과물:**
- 좌우 분할 레이아웃
- 실시간 마크다운 → HTML 변환
- 코드 구문 강조 (highlight.js 등)
- 파일 다운로드 기능

**실습 포인트:**
- marked.js 같은 라이브러리 활용법 학습
- "툴바에 볼드/이탤릭/링크 버튼 추가해줘"
- XSS 방지를 위한 sanitize 처리 확인

**확장 아이디어:**
- HTML/PDF 내보내기
- 자동 저장 (localStorage)

### 시나리오 10: QR 코드 생성기

**목표:** 텍스트나 URL을 입력하면 QR 코드를 생성하고 다운로드할 수 있는 앱을 만든다.

> "QR 코드 생성기를 만들어줘. URL이나 텍스트를 입력하면 QR 코드가 실시간으로 생성되고, PNG로 다운로드할 수 있게 해줘. QR 코드 색상과 크기도 조절 가능하게."

- qrcode.js 라이브러리 활용
- "로고를 QR 코드 중앙에 넣을 수 있게 해줘"

**확장 아이디어:**
- 대량 QR 코드 생성 (CSV 입력)

### 시나리오 11: 날씨 앱

**목표:** OpenWeatherMap API를 연동하여 도시별 날씨 정보를 보여주는 앱을 만든다.

**실습 프롬프트 예시:**
> "날씨 앱을 만들어줘. 도시 이름을 검색하면 현재 온도, 습도, 풍속, 날씨 아이콘을 보여줘. OpenWeatherMap API를 사용하고, 5일 예보도 카드 형태로 보여줘."

**사전 준비:**
- OpenWeatherMap 무료 API 키 발급 (https://openweathermap.org/api)

**기대 결과물:**
- 도시 검색 기능
- 현재 날씨 + 5일 예보
- 날씨 아이콘 표시
- 로딩/에러 상태 처리

**실습 포인트:**
- fetch API와 async/await 패턴 학습
- "현재 위치 기반으로 자동 검색되게 해줘" (Geolocation API)
- API 키를 환경 변수로 관리하는 방법 질문해보기
- 에러 핸들링 (잘못된 도시명 입력 시)

**확장 아이디어:**
- 최근 검색 도시 목록
- 날씨에 따른 배경 이미지 변경

### 시나리오 12: 영화 검색 앱

**목표:** TMDB 또는 OMDB API로 영화를 검색하고 상세 정보를 보여주는 앱을 만든다.

**실습 프롬프트 예시:**
> "영화 검색 앱을 만들어줘. OMDB API를 사용해서 영화 제목으로 검색하면 포스터, 제목, 연도, 평점, 줄거리를 카드로 보여줘. 카드를 클릭하면 상세 모달이 뜨게 해줘."

**사전 준비:**
- OMDB API 키 발급 (https://www.omdbapi.com/)

**기대 결과물:**
- 검색 입력 + 결과 그리드
- 영화 카드 (포스터, 제목, 연도)
- 상세 모달 (줄거리, 평점, 출연진)

**실습 포인트:**
- API 응답 데이터 구조 파악
- "검색 결과 페이지네이션 추가해줘"
- "즐겨찾기 기능 추가해줘 (localStorage)"
- 디바운스 검색 구현 요청

**확장 아이디어:**
- 장르별 필터
- 추천 영화 목록

### 시나리오 13: 포모도로 타이머

**목표:** 25분 집중 / 5분 휴식 사이클의 포모도로 타이머를 만든다.

**실습 프롬프트 예시:**
> "포모도로 타이머를 만들어줘. 25분 집중 → 5분 휴식 → 반복 사이클이고, 4사이클 후 15분 긴 휴식이야. 원형 프로그레스 바로 남은 시간을 시각화하고, 타이머 종료 시 알림음이 울리게 해줘."

**기대 결과물:**
- 원형 프로그레스 바 타이머
- 집중/휴식 모드 자동 전환
- 알림음 (Web Audio API 또는 Audio 태그)
- 사이클 카운터

**실습 포인트:**
- CSS로 원형 프로그레스 바 구현 방법 학습
- "집중 시간과 휴식 시간을 커스텀할 수 있게 해줘"
- Notification API로 브라우저 알림 추가 요청
- 타이머 정확도 이슈 (setInterval drift) 토론

**확장 아이디어:**
- 일별/주별 집중 시간 통계
- 작업별 시간 추적

### 시나리오 14: 간단한 메모장 앱

**실습 프롬프트 예시:**
> "메모장 앱을 만들어줘. 왼쪽에 메모 목록, 오른쪽에 편집 영역이 있는 레이아웃이야. 메모 추가/삭제/수정이 가능하고, 자동 저장되게 해줘. 메모 검색 기능도 넣어줘."

**기대 결과물:**
- 사이드바 + 편집 영역 레이아웃
- CRUD 기능
- 자동 저장 (localStorage)
- 검색/필터

**실습 포인트:**
- "메모에 태그를 추가할 수 있게 해줘"
- "마크다운 지원 추가해줘"
- 데이터 구조 설계 (배열 vs 객체)
- contentEditable vs textarea 차이 이해

**확장 아이디어:**
- 메모 내보내기/가져오기 (JSON)
- 폴더 구조로 정리

### 시나리오 15: GitHub 프로필 뷰어

### 바이브코딩 효과를 극대화하는 프롬프트 작성법

1. **구체적으로 요청하기**: "웹사이트 만들어줘" 보다 "React로 다크 모드 지원하는 포트폴리오 사이트 만들어줘. Hero 섹션에 타이핑 애니메이션 넣어줘"가 훨씬 좋은 결과를 냄
2. **점진적으로 확장하기**: 한 번에 모든 기능을 요청하지 말고, 기본 기능 → 추가 기능 → 개선 순서로 요청
3. **Kiro Spec 활용하기**: 중급 이상 시나리오는 Spec으로 요구사항/설계/태스크를 먼저 정리하면 훨씬 체계적
4. **코드 이해하기**: 생성된 코드를 반드시 읽고, 이해 안 되는 부분은 "이 부분 설명해줘"로 질문
5. **의도적으로 깨뜨리기**: 코드를 일부러 수정해서 에러를 만들고, Kiro에게 디버깅 요청해보기
